# case_prompter.py — документация

Скрипт для тестирования системных промптов из JSON-файлов с разными AI-моделями и параметрами.

## Запуск

```bash
python case_prompter.py
```

## Что делает скрипт

1. Читает все JSON-файлы из папки `Prompts/`
2. Пользователь выбирает один или несколько промптов (или все сразу)
3. Пользователь выбирает провайдера и модель
4. Пользователь задаёт temperature и max_tokens
5. Для каждого промпта — вводит вопрос (или использует `test_input` из JSON)
6. Отправляет запрос через `openai_client.py`
7. Выводит ответ в терминал с метаданными (токены, стоимость, параметры)
8. Сохраняет результат в `cases/case_<prompt_id>_<YYYYMMDD_HHMMSS>.md`
9. Предлагает повторить тот же промпт с другими параметрами или моделью

## Структура папок

```
.
├── case_prompter.py        # скрипт
├── Prompts/                # системные промпты в JSON
│   ├── 01_summary_prompt.json
│   ├── 02_code_structure.json
│   └── 03_task_planning_prompt.json
└── cases/                  # результаты (создаётся автоматически)
    └── case_summary_20260324_143000.md
```

## Формат JSON-промпта

Скрипт читает следующие поля из JSON:

| Поле | Обязательное | Описание |
| --- | --- | --- |
| `prompt_id` | да | Уникальный идентификатор (используется в имени файла) |
| `name` | да | Название промпта (отображается в меню) |
| `category` | нет | Категория (отображается в меню) |
| `version` | нет | Версия промпта |
| `description` | нет | Краткое описание |
| `role` | нет | Роль AI-ассистента |
| `context` | нет | Контекст задачи |
| `structure` | нет | Структура ожидаемого ответа |
| `format` | нет | Требования к формату ответа |
| `test_input` | нет | Пример вопроса для быстрого тестирования |

Системное сообщение собирается по схеме: **роль → контекст → структура → формат**.

## Пример JSON-промпта (минимальный)

```json
{
  "prompt_id": "my_prompt",
  "name": "Мой промпт",
  "role": "Ты — эксперт в области ...",
  "context": "Пользователь хочет ...",
  "test_input": "Пример вопроса для тестирования"
}
```

## Формат выходного Markdown-файла

Каждый файл в `cases/` содержит:

- Параметры запуска (промпт, провайдер, модель, temperature, max_tokens)
- Статистику токенов (prompt_tokens, completion_tokens, total_tokens)
- Стоимость запроса в ₽
- Полный системный промпт (как он был отправлен модели)
- Вопрос пользователя
- Ответ модели

## Провайдеры и модели

Используются те же провайдеры, что и в остальных скриптах проекта (из `config.py`):

| Провайдер | Модели |
| --- | --- |
| Z.AI | GLM-4.7-Flash (free), GLM-4.5-Flash (free), GLM-4.7, GLM-4.5, GLM-5 |
| ProxyAPI | GPT-4.1 Nano, GPT-4.1 Mini, GPT-4.1, GPT-4o Mini, GPT-4o, GPT-3.5 Turbo |
| GenAPI | GPT-4.1 Mini, GPT-4.1, GPT-4o, Claude Sonnet 4.5, Gemini 2.5 Flash, DeepSeek Chat, DeepSeek R1 |

## Параметры модели

| Параметр | Описание | Рекомендация |
| --- | --- | --- |
| Temperature | Степень случайности ответа. `0.0` — детерминированный, `2.0` — максимально творческий | `0.2–0.7` для структурированных ответов |
| Max tokens | Максимальная длина ответа в токенах | `1024` по умолчанию |

Скрипт предупреждает при temperature > 1.5 и запрашивает подтверждение.

## Стоимость запросов

Стоимость рассчитывается по формуле:

```
стоимость = (prompt_tokens / 1000 × price_in) + (completion_tokens / 1000 × price_out)
```

Тарифы берутся из `config.py` (поля `price_in`, `price_out` в ₽ за 1К токенов).
Для бесплатных моделей Z.AI стоимость = 0.

## Тестирование одного промпта с несколькими моделями

После получения ответа скрипт предлагает:
- Повторить тот же промпт и вопрос с другими параметрами (temperature, max_tokens)
- Или сменить провайдера/модель полностью

Это позволяет сравнивать качество ответов разных моделей на один и тот же промпт.

## Зависимости

```
openai_client.py   — единый HTTP-клиент для всех провайдеров
config.py          — провайдеры, модели, цены, API-ключи
.env               — секреты (ZAI_API_KEY, PROXY_API_KEY, GEN_API_KEY)
```
